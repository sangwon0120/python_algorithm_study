## 스터디 방식 
  1. 스터디 전까지 주어진 파트를 공부하고 문제를 풀어 각자 branch에 push한다.
  2. 스터디 시 각자 푼 코드를 설명하며 풀이 방식, 어려웠던 점 등을 발표한다.
  3. 발표자를 제외한 팀원들은 발표자의 code를 review한다.
